Trindex

Persistent Semantic Memory for AI Agents

via Model Context Protocol (MCP)

What is Trindex?

  • Standalone Go binary for AI agent memory
  • Vector embeddings in PostgreSQL + pgvector
  • Semantic similarity search
  • Hybrid search: vector + full-text (RRF)
  • Namespace scoping with global fallback
  • REST API + Web UI + CLI

The Problem

AI Agents Have No Memory

  • Every conversation starts fresh
  • Context windows are limited
  • No persistence across sessions
  • Building custom memory is complex
Existing solutions: Python-only, heavy deps, or low-level

Key Features

🤖 MCP-Native

Works with Claude Code, opencode, Cursor

🔍 Semantic Search

pgvector + tsvector + RRF fusion

📦 Namespaces

Isolated spaces with global fallback

🌐 Multi-Interface

REST API, Web UI, CLI

./trindex memories list --namespace work
./trindex memories create --content "..."
./trindex search "architecture" --top-k 10

# Data Management
./trindex export --output backup.jsonl
./trindex import memories.jsonl

Database Schema

CREATE TABLE memories (
    id          UUID PRIMARY KEY,
    namespace   TEXT NOT NULL DEFAULT 'default',
    content     TEXT NOT NULL,
    embedding   VECTOR(768),
    metadata    JSONB DEFAULT '{}',
    search_vec  TSVECTOR GENERATED ALWAYS AS 
                  (to_tsvector('english', content)) 
                  STORED,
    created_at  TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX memories_embedding_hnsw_idx
    ON memories USING hnsw 
    (embedding vector_cosine_ops);

Getting Started

# Clone and setup
git clone https://github.com/dbehnke/trindex.git
cd trindex
cp .env.example .env
# Edit .env with your embedding endpoint

# Run with Docker Compose
docker compose up -d

# Or build locally
go build -o trindex ./cmd/trindex
./trindex doctor  # Verify setup
